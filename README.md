# 🛡️ File Guardian — Personal Data Encryption System

> **A military-grade file encryption tool disguised as a harmless image file.
> Designed for professionals who need to protect confidential data instantly
> in emergency situations — no interaction required.**

---

## 📌 What Is This?

**File Guardian** is a personal data security system built for one purpose —
**protecting your confidential files from unauthorized access instantly.**

When you suspect someone is trying to steal your data, you simply
double-click what looks like a normal photo on your desktop.
In the background, your files are silently encrypted using
AES-128 military-grade encryption while a real photo opens
in Windows Photos as a decoy — leaving the intruder completely unaware.

You monitor the entire process live from your phone via a web dashboard,
and the tool **deletes itself** after completing encryption, leaving zero trace.

## ⚙️ How It Works

### 🔒 Encryption Flow

```
User double-clicks disguised exe
        │
        ├─→ Windows Photos opens (shows real image as decoy)
        │
        └─→ File Guardian runs silently in background
                │
                ├─→ Scans target folder / all drives
                ├─→ Encrypts every file with AES-128 + your password
                ├─→ Pushes live progress to ntfy.sh every 6 seconds
                ├─→ You monitor from phone dashboard
                └─→ Tool self-deletes after completion
```

### 🔓 Decryption Flow

```
Press Ctrl + Shift + G
        │
        └─→ Hidden UI appears
                │
                └─→ Enter password → click Unlock
                        │
                        └─→ All files restored to original state
```

### 🔐 Encryption Details

| Property | Value |
|---|---|
| Algorithm | AES-128 (Fernet) |
| Key Derivation | PBKDF2-HMAC-SHA256 |
| Iterations | 390,000 |
| Salt | Unique 16-byte random salt per folder |
| Password Storage | SHA-256 hash only — real password never stored |
| Encrypted extension | `.guardian` added to each file |

## 🛠️ Components

### 1. Guardian Builder (Control Panel)
> `guardian_builder.py` / `GuardianBuilder.exe`

A GUI tool to configure and build your custom disguised `.exe`.

**What you configure:**
- 📁 Target paths to encrypt (specific folders or entire drives)
- 🖼️ Decoy image (any JPG/PNG — becomes the exe icon too)
- 🔑 Encryption password
- 📋 File types to encrypt (Documents, Images, Adobe, 3D/CAD, etc.)
- ⚙️ Whether to include system files/folders (extreme mode)
- 📡 ntfy.sh topic for live monitoring
- 🏷️ Output exe name (e.g. `holiday_photo.jpg`)

**What it outputs:**
- A single disguised `.exe` file ready to deploy

---

### 2. File Guardian (The Encrypted Tool)
> Auto-generated by Guardian Builder

The actual encryption engine. Disguised as a photo file.

**Behaviour when double-clicked:**
1. Opens decoy image in Windows Photos
2. Runs completely hidden in background
3. Waits 3 seconds then starts encrypting
4. Pushes live progress to your dashboard
5. Self-deletes after encryption completes

**Press `Ctrl + Shift + G`** to reveal the hidden UI for manual control and decryption.

---

### 3. Live Dashboard
> `https://madhan-shan.github.io/guardian-monitor/`

A GitHub Pages website that shows real-time encryption progress.
Powered by **ntfy.sh** — no tokens, no accounts needed.

**Shows:**
- Current status (Encrypting / Complete / Error)
- Progress percentage and file count
- Current file being encrypted
- Target drives
- Start time and last update time
- Refreshes every 5 seconds automatically

## 📋 File Types Encrypted

### ✅ Always Encrypted
| Category | Extensions |
|---|---|
| Documents | `.doc .docx .pdf .txt .xls .xlsx .ppt .pptx .rtf .odt .csv` |
| Images | `.jpg .png .gif .bmp .tiff .raw .cr2 .nef .arw .dng .heic .webp` |
| Videos | `.mp4 .avi .mkv .mov .wmv .flv .m4v .mpg .3gp .webm` |
| Audio | `.mp3 .wav .flac .aac .m4a .ogg .wma .aiff .opus` |
| Archives | `.zip .rar .7z .tar .gz .bz2 .iso .dmg` |
| Databases | `.db .sqlite .mdb .accdb .sql .dbf` |
| Adobe Suite | `.psd .ai .indd .eps .prproj .aep .xd .lrcat .psb .abr` |
| 3D / CAD | `.dwg .dxf .skp .blend .fbx .obj .stl .max .c4d .mb .ma` |
| Dev Files | `.java .cs .cpp .php .rb .go .ts .jsx .env .pem .key .crt .yml` |
| Email | `.pst .ost .eml .msg .mbox` |

### ❌ Always Skipped (keeps PC bootable)
| Category | Extensions / Folders |
|---|---|
| Executables | `.exe .dll .sys .drv .ocx .scr .msi` |
| Scripts | `.bat .cmd .lnk .ini .py .vbs .ps1` |
| System folders | `Windows, Program Files, AppData, ProgramData` |
| Recovery | `$Recycle.Bin, System Volume Information, Boot` |

### ⚠️ Optional (extreme situations only)
- System files (`.exe .dll .sys`) — **PC may not boot after!**
- System folders (Windows, Program Files) — **extreme only!**

## ⚠️ Important Warnings

> **THERE IS NO PASSWORD RECOVERY.**
> If you forget your password, your files cannot be decrypted — ever.
> Write your password somewhere physically safe.

> **Keep a copy of the tool on a USB drive.**
> Since the exe self-deletes, always have a backup to decrypt later.

> **Test before relying on it.**
> Always test encrypt + decrypt on a small test folder before using in production.

> **Dashboard is public.**
> The ntfy.sh topic is semi-public. Don't use sensitive filenames — only progress % is shown.
